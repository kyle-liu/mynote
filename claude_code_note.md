## 使用Cluade Code 编程思维的转变
之前使用AI编程工具的时，我都是在其他AI工具里面让AI写一个类或者函数等代码片段，然后通过复制-粘贴的方式把代码粘贴到IDE。养成这种习惯的根本原因还是对目前AI编程工具的能力上限理解不够，一定要摒弃这种习惯。
经过使用Cluade Code从零到交付实战一个项目，这个项目从需求分析、UI设计、系统设计、前后端代码coding,我一句话文档和一行代码都没有写。 使用Cluade Code我们应该把它当成一个非常有经验且不知疲倦的编程伙伴，
引用吴恩达 Claude Code 笔记精华版 https://mp.weixin.qq.com/s/Of1qAQuAQDrkyWIZ0ug-dw：
* 范式转变：从“指令式工具”到“智能体伙伴”：核心在于思维模式的跃迁。你需要将 Claude Code 视为一个能够自主规划、执行并进行反思的初级合伙人，而非一个被动的代码生成器。
* 上下文工程：对话的本质是信息架构：与 AI 的每一次交互，都是一次微型的信息架构设计。成功的关键在于如何精准、高效地构建上下文，这决定了其输出质量的上限。
* 持久化记忆：构建项目的“第二大脑”：通过 CLAUDE.md 系统，我们将为 AI 构建一个可版本化、可共享的项目知识库，实现跨会话、跨团队的知识沉淀与复用。
* 工程化整合：将 AI 注入开发生命周期：真正的价值在于将 AI 无缝融入成熟的工程实践中，例如测试驱动调试（TDD）、并行开发（Git Worktree）和持续集成/持续部署（CI/CD）。
* 元编程思维：从“使用 AI”到“编排 AI”：通过 MCPs 和 Hooks 等高级功能，开发者将从 AI 的“使用者”转变为 AI 工作流的“设计者与编排者”，实现开发流程的深度自动化与定制。

## 什么是好的Prompt(case来源于Claude Code官方最佳实践)
### 模糊 vs. 具体
#### case1 差的指令: “为 foo.py 添加测试”
* 问题: 这个指令太模糊了。要添加什么样的测试？单元测试还是集成测试？需要覆盖哪些功能或场景？执行者完全不知道从何下手。
* 好的指令: “为 foo.py 编写一个新的测试用例，覆盖用户未登录的边界情况。避免使用 mock。”
* 改进点:
  * 明确的目标: “编写一个新的测试用例”。
  * 具体的场景: “覆盖用户未登录的边界情况”，指明了测试的核心。
  * 清晰的约束: “避免使用 mock”，给出了技术实现的限制，避免了不必要的方案。
### 情绪化抱怨 vs. 可执行任务
#### case2 差的指令: “为什么 ExecutionFactory 的 api 这么奇怪?”
* 问题: 这是一个带有情绪的抱怨或一个开放式问题，而不是一个任务。它没有要求对方做任何具体的事情，无法推动问题解决。
* 好的指令: “查阅 ExecutionFactory 的 git 历史，并总结其 api 是如何演变而来的。”
* 改进点:
  * 将问题转化为行动: 把“为什么”的困惑，转变成了“去调查”的具体动作。
  * 指明调查路径: “查阅 git 历史”提供了解决问题的明确方法。
  * 定义交付成果: “总结其 api 是如何演变而来的”，要求一个清晰的、可交付的结果。
### 缺乏上下文 vs. 提供完整上下文和路径
#### case3 差的指令: “添加一个日历小部件”
* 问题: 这个指令信息严重不足。小部件要实现成什么样？有什么功能要求？技术上有什么规范？执行者需要反复沟通才能明确需求。
* 好的指令:
  *  “查看主页上现有小部件的实现方式，以理解其模式...” -> 提供学习路径
  * “HotDogWidget.php 是一个很好的起点” -> 提供具体参考范例
  * “...让用户可以选择月份并向前/向后翻页选择年份” -> 明确功能需求
  * “从头开始构建，除了代码库中已使用的库外，不要使用其他库” -> 设定技术约束
改进点: 这条指令几乎是一个小型项目计划。它告诉执行者做什么 (What)、为什么这么做 (Why - 遵循现有模式)，以及如何做 (How - 参考范例、遵循约束)。

## CLAUDE.md 构建知识库
### CLAUDE.md 文件的作用
由于Cluade Code 本身没有存储机制，CLAUDE.md 的分层加载机制允许我们清晰地分离团队规范与个人偏好。加载顺序为：企业策略 -> 用户记忆 -> 项目记忆，更具体的配置会覆盖或补充上层配置
Claude Code 会从当前工作目录开始，向上递归查找 CLAUDE.md 文件，直到根目录。 [11] 此外，CLAUDE.md 支持 @path/to/file.md 语法导入其他文件，这已取代了现已废弃的 CLAUDE.local.md
